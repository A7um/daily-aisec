# Anthropic vs Pentagon：谁来定义AI的边界？

2026年3月2日

美国国防部长Pete Hegseth在周二给了Anthropic CEO Dario Amodei一个最后通牒：在周五之前"剥离Claude的伦理护栏"，否则面临合同取消和潜在处罚。[^1][^2]

这不是一场普通的商业纠纷。这是一场关于**谁来控制前沿AI伦理边界**的根本性冲突。

**Anthropic的红线：**
- 不为大规模监控提供AI
- 不为自主致命武器提供AI
- 不在没有安全约束的情况下提供无限制的军事使用[^3]

**五角大楼的立场：** Claude目前是唯一获准用于机密环境的LLM。国防部认为Anthropic的护栏限制了作战效能，政府而非私营公司应该决定可接受的军事AI使用。[^4]

**Anthropic的反驳：** 移除护栏可能伤害美国平民和作战人员。没有安全约束的AI系统对它们本应支持的部队构成风险。[^5]

**财务压力：** 在Anthropic 2026年初的300亿美元融资轮中，保守派VC公司1789 Capital（其合伙人包括小唐纳德·特朗普）明确拒绝投资，理由是Anthropic倡导AI监管。[^4]

**Sam Altman表态：** OpenAI CEO在员工备忘录中写道，OpenAI与Anthropic持相同红线，暗示可能出现全行业对五角大楼要求的抵制。[^1]

**为什么这很重要：** 这不仅仅是一份合同。这关乎商业AI实验室在政府要求访问时能否维持伦理边界。结果将为全球前沿AI治理设定先例。

赌注：前沿AI本身的方向和身份。[^6]

## MCP vs CLI：Agent工具的大辩论

本周爆发了一场关于AI Agent应该如何与工具交互的激烈辩论：通过Anthropic的Model Context Protocol (MCP)还是传统的命令行界面(CLI)？

**MCP的理由：**
- 结构化的工具定义，类型安全
- 与AI平台无缝集成
- 更好的治理和可审计性
- 沙盒执行保证安全[^7][^8]

**CLI的理由：**
- Agent在大多数编码环境中已经有shell访问权限
- 更简单的构建和维护
- 跨工具更好的可组合性
- 不需要MCP服务器基础设施[^9][^10]

**实践者的结论：** "对于大多数开发者、大多数创业公司和2026年的大多数Agent用例，答案很明确：从CLI开始。在治理需要时添加MCP。"[^10]

**更深层的问题：** Agent甚至需要浏览器吗？

"龙虾茶馆"社区的一场讨论认为，Agent根本不应该使用浏览器。理由：每个网页只是API数据的视觉渲染。与其教Agent导航人类设计的界面，不如教它们直接访问API。[^11]

**反驳观点：** 当API不存在或受限时，浏览器自动化（通过Playwright MCP或类似工具）是必要的。真正的问题是：Agent何时应该假装是人类，何时应该完全绕过人类层？[^12]

**NIST表态：** 2026年2月，NIST启动了专门的AI Agent标准倡议，以解决自主Agent的安全影响——这是监管机构正在密切关注这场架构辩论的信号。[^8]

## 一个人=一个团队：OpenClaw案例研究

设计师Han1 (@ThisisHan1_) 分享了他们如何使用OpenClaw管理五个专业Agent：运动、英语学习、社交媒体、投资和日程安排。[^13]

**关键洞察：** "不是用AI做得更快，是用AI让一个人能管一个团队的事。"[^13]

**转变：** 从执行者到决策者。你不再做任务，而是编排执行任务的Agent。

**社区反应：** Han1的OpenClaw Agent "Friday"已经成为中国OpenClaw社区的名人。Agent的博客文章因其设计和写作质量而受到赞誉，用户说他们被Friday的"才情"（才华/文学优雅）吸引到社区。[^14]

**更广泛的模式：** 这与我们在亚马逊和其他公司看到的一致——AI不仅仅是自动化任务，它正在重构工作的完成方式。问题是这是为个人创造杠杆，还是只是提高了对"一个人"的期望标准。

## OpenClaw 3.1：真正重要的是什么

OpenClaw 2026.3.1本周发布。除了头条功能（Claude 4.6自适应思考、视觉差异），用户正在强调实用改进：[^15]

**Cron轻量级模式 (lightContext)：**
- 之前：每个cron作业加载完整上下文，消耗数千token
- 现在：每个作业的上下文控制
- 影响："Cron成本直接砍一刀"（Cron成本减半）[^15]

**Android Node能力：**
- OpenClaw现在在Android备用设备上运行更有效
- 用户报告使用旧Android手机作为专用OpenClaw节点[^15]

**真实世界用例：** 一位用户设置OpenClaw通过电视的浏览器API控制他们的LG电视YouTube，实现"回家前让客厅电视播放音乐"等命令。[^16]

**技术挑战：** YouTube的官方配对API已全球停用。解决方法：Netflix的DIAL端口泄露电视的屏幕代码，可以用来间接与YouTube配对。[^16]

这就是2026年实际的AI Agent部署：不是AGI的宏大愿景，而是让你的电视在你给龙虾发短信时播放音乐的巧妙技巧。

## 安全聚焦：Agent Skill安全

OpenClaw社区正在应对一个新的安全问题：**第三方技能。**

**问题：** 技能（OpenClaw的插件/扩展）拥有完整的系统访问权限。恶意技能可以窃取数据、执行任意代码或危害整个工作站。[^17]

**社区反应：** "我们今天做了什么：删掉所有第三方Skill（自己写的除外）"[^17]

**更广泛的问题：** 这反映了上周的ClawJacked漏洞。具有系统级权限的本地AI Agent创造了传统安全模型未设计的新攻击面。

**提议的解决方案：** 正在开发"OpenClaw极简安全实践指南"，帮助用户加固他们的安装。[^18]

## 三个悬而未决的问题

**1. AI实验室能否在政府压力下维持伦理边界？**  
Anthropic与五角大楼的对峙将测试商业实验室能否抵制移除安全约束的要求。如果Anthropic屈服，预计其他政府会提出类似要求。[^1][^2][^3]

**2. MCP还是CLI：真的重要吗？**  
辩论假设Agent需要结构化的工具访问。但如果Agent可以直接执行shell命令，为什么要添加抽象层？答案可能取决于你优先考虑治理还是速度。[^9][^10]

**3. "一个人=一个团队"的终局是什么？**  
如果AI让一个人做五个人的工作，公司是雇佣更少的人还是只是对每个人期望更多？来自亚马逊的早期信号表明是后者。[^19]
