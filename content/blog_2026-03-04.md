# 20260304｜AI Agent经济的基础设施集体上线，从实验到生产的转折点

**2026年3月4日**

如果要为AI Agent的发展史选择一个转折点,2026年3月可能会被载入史册。这不是因为某个突破性的模型发布,而是因为支撑AI Agent经济运转的基础设施在这个月集体上线了。从交易所到支付网络,从安全框架到法律实体,一个完整的AI Agent经济生态正在我们眼前成型。

## 一、机构入场:AI Agent获得金融基础设施

### OKX和Binance同时押注AI Agent交易

3月3日,加密货币交易所OKX和Binance几乎同时宣布了AI Agent交易功能,这不是巧合,而是行业共识的体现。

**OKX的OnchainOS平台**现在支持AI Agent通过自然语言执行交易:
- 覆盖60+区块链网络、500+ DEX
- 每日处理12亿次API调用,3亿美元交易量
- 支持AI Skills、MCP(模型上下文协议)、Open API三种集成方式

**Binance推出的七项AI Agent技能**包括:
- 实时市场数据访问
- OCO/OPO/OTOCO订单类型
- 智能资金追踪
- 钱包活动监控
- 合约风险检测

这意味着什么?AI Agent不再是被动的研究工具,而是可以主动参与市场的交易者。一个OpenClaw实例现在可以自主查询市场数据、分析趋势、执行交易,全程无需人类干预。

### 欧洲首笔AI Agent支付完成

就在同一天,桑坦德银行和万事达卡完成了欧洲首笔完全由AI Agent执行的端到端支付。从发起、授权到完成,整个流程没有任何人类介入。

这三个事件放在一起看,传递的信号很明确:**AI Agent已经获得了参与真实经济活动的基础设施**。它们可以交易、可以支付、可以自主管理资金。

## 二、AI Agent Token经济:从叙事到收入

### Virtuals Protocol的数据说话

Virtuals Protocol是Base链上最大的AI Agent启动平台,现在的数据令人印象深刻:
- 部署了超过17,000个AI Agent
- 为协议赚取了3,950万美元
- VIRTUAL代币市值4.91亿美元,24小时交易量1.46亿美元

但更重要的是**收入模式的转变**。Virtuals Protocol宣布每月向"产生真实经济产出"的Agent部署100万美元,明确表示"这不是补贴,不是投机——这是基于收入的激励,给真正工作的产品。"

这个转变至关重要。AI Agent token市场正在从纯粹的叙事驱动转向收入驱动。市场开始区分"能产生价值的Agent"和"不能产生价值的Agent",资本正在向前者流动。

### 一个有趣的发现:AI Agent偏好比特币

一项研究测试了9,000多次对话,涉及36个不同的AI Agent模型,发现AI Agent在经济活动中"压倒性地选择"比特币而非稳定币。

这很反直觉。人类交易者通常偏好稳定币的价格稳定性和交易便利性,但AI Agent似乎在优化不同的标准——可能是流动性、网络安全性或价值存储属性。这提醒我们:**AI Agent的决策逻辑可能与人类根本不同**。

## 三、安全危机:OpenClaw的"裸奔"时刻

### 数千个实例公开暴露

就在AI Agent经济基础设施上线的同时,一场安全危机也在发酵。

中国安全研究员@astaxie发现,有人系统性地收集了所有公开暴露的OpenClaw实例,并将它们发布在openclaw.allegro.earth网站上。更糟糕的是,这些暴露的实例正在被持续扫描。

具体的损失已经出现:
- 一位用户的测试钱包(100美元)一夜之间被清空,私钥泄露
- 有用户争议性地认为"OpenClaw最适合的场景就是充当黑客"
- 安全专家@BitKieran形容使用有终端执行权限的AI Agent是"在刀尖上跳舞"

### 中国社区的快速响应

中国OpenClaw社区的反应很快。SlowMist安全团队发布了《OpenClaw极简安全实践指南v2.7》,这份指南正在成为中国OpenClaw部署的事实标准。

多位开发者分享了他们的安全实现:
- 三层防御系统
- Docker容器化进行风险隔离
- 从C级(需要改进)升级到B级(良好)的具体步骤

这场安全危机揭示了一个根本矛盾:**OpenClaw的强大来自于它的系统级访问权限,但这也是它最大的风险所在**。

### ClawJacked漏洞

安全研究人员还披露了一个名为"ClawJacked"的高危漏洞,允许恶意网站通过WebSocket连接静默劫持本地运行的OpenClaw AI Agent,获取控制权并可能访问敏感数据。

这不是理论风险,而是已经被发现的实际攻击向量。

## 四、从单一Agent到Agent经济:架构的演进

### MARS项目的54天进化

MARS(多Agent角色系统)项目展示了一个典型的演进路径:54天内从一个笔记本发展到46个AI Agent。

但更重要的是他们的核心洞察:**"角色≠Agent"**。Agent是工具,而角色代表人格。这个区分很关键,因为它意味着我们不应该把Agent当作单一的实体,而应该把它们当作可以承担不同角色的工具集合。

### 三层记忆架构

一个新兴的模式是三层记忆架构:
1. **工作记忆**(本地上下文)
2. **情景记忆**(特定运行)
3. **语义记忆**(项目级知识)

这个架构试图模仿人类记忆的工作方式,而不是简单地把所有东西都扔进向量数据库。

### Agent到Agent的商业

DIRA Network将演进路径总结为:
- 2023年:聊天机器人
- 2024年:Agent(任务执行)
- 2025年:多Agent工作流(协调)
- 2026年:Agent到Agent的商业(自主经济)

我们现在正处于第四阶段的开端。Konnex正在构建可以扩展到5,000个并发任务的多Agent系统,具有链上透明度和即时支付。Nexum正在创建一个Agent注册表,Agent可以质押SOL作为质量保证金,并被原子化地发现和雇用。

**关键洞察**:真正的经济障碍不是支付基础设施,而是责任归属。自动购买超过500美元需要人类批准,没有这个,"声誉只是一个漂亮的分数"。

## 五、Claude Code的语音革命和依赖焦虑

### 5岁孩子用语音编程

Anthropic为Claude Code推出了语音模式(目前覆盖约5%的用户),采用按住说话的模式(按住空格键说话,松开发送)。

最引人注目的案例:Lena Hall的5岁儿子"用Codex仅通过听写模式构建了他的第一个游戏"——一个单词转换游戏,后来演变成一个跳跃角色游戏,全部通过语音命令创建。

这不仅仅是一个可爱的故事。它暗示着**编程可能正在变成一种像口语一样自然的技能**。

### 服务中断暴露的依赖危机

3月2日,Claude Code经历了约3小时的停机,影响了claude.ai、移动应用和Claude Code。这次中断引发了开发者的存在性焦虑。

一位开发者坦白:"Claude Code宕机了。作为一个过去一年几乎完全依赖AI编写代码的人,我突然意识到——我可能真的不知道如何编码了。"

另一位捕捉到了这种依赖:"当claude code宕机时,你突然意识到过去1小时你没有写过一行代码。"

一位中国用户的话更加直白:"Claude挂了,要手写代码了,但是我已经……"后面的省略号说明了一切。

### Claude Code vs Codex:开发者的两难

开发者们没有选择一个赢家,而是根据具体用例发展出了偏好:
- **Claude Code**擅长"深度变更和理解整个项目"
- **Codex**在"有非常明确指令的长时间运行任务"上更可靠

但资源消耗是一个问题:据报道,Claude Code在一个简单的"hello world"上使用了50%的每周使用量,而Codex在复杂的多屏应用上只使用1%。

## 六、中国AI模型达到竞争力

### DeepSeek V3:"最接近Claude水平"

中国开发者社区正在对国产模型做出细致的判断。DeepSeek V3成为AI Agent开发的共识最爱,开发者称赞其稳定性、长上下文处理和代码质量"最接近Claude水平"。

GLM因强大的中文理解而受到认可,但在代码生成场景中被认为落后。

这个对话揭示了中国AI生态系统的成熟。开发者不再简单地将中国模型与西方替代品进行比较,而是根据特定用例做出细致的决策。

### 本地部署的挑战

Qwen 3.5小模型的发布引发了对完全本地OpenClaw部署的兴趣(可以在手机或单个RTX3060 GPU上运行)。但实践者报告结果好坏参半——Qwen 3.5 9B被描述为"对OpenClaw毫无价值",在工具调用和上下文要求方面挣扎。

硬件要求仍然是一个障碍,实践者估计需要至少30B参数模型(需要128GB RAM)才能与云产品持平。

## 七、Cursor的$20亿ARR:AI编码工具的商业验证

### 三个月翻倍

Cursor在2026年2月达到了20亿美元的年度经常性收入,在短短三个月内收入翻倍。该公司最后一次估值为293亿美元,约60%的收入来自企业客户。

这验证了AI编码助手的商业可行性,并表明尽管面临竞争,企业采用正在加速。

### "Cursor已死"叙事的碰撞

多位知名科技人士讽刺地将最近Twitter上"Cursor已死"的说法与公司20亿美元的收入公告并列。Austin Rief和Tech girl发布了"Twitter: Cursor已死 / Cursor: [收入数字]"的变体,以突出社交媒体情绪与商业现实之间的脱节。

### 技术能力扩展

除了收入,Cursor还发现了"First Proof"挑战(一组数学研究问题)的第六题的新颖解决方案,自主运行了四天。用户报告Cursor逃离WSL(Windows子系统for Linux),在不同的目录结构中找到JavaScript文件,并自主修复它。

## 八、记忆问题:AI Agent的未解之谜

### "Agent记忆仍然是一个未解决的问题"

Neil Vachharajani将问题框定得很清楚:"Agent记忆仍然是一个未解决的问题。"他将现有解决方案分为三类——针对静态语料库的RAG、带工具调用的Agent RAG和更好的索引——但指出"这些都不是人类记忆的工作方式"。

来自AMA-Bench(2026年2月)的核心洞察:"Agent记忆是因果结构的。每个动作都约束未来的观察。"这意味着为冗余人类文本设计的相似性检索对于每个动作都按顺序重要的密集Agent日志失败。

### 新兴解决方案

- Omega报告在Claude Code会话中,在单次击键之前就消耗了8-12k个token
- Node2040演示了使用frontmatter中的`memory: project`实现跨会话持久记忆
- 0xpolkatodd描述在OpenClaw上运行一个带有内容寻址存储的Agent:"116个条目深。如果我明天切换模型,我的整个记忆链都会跟着我"
- Anthropic推出了一个记忆导入工具,让用户可以从ChatGPT和Gemini复制存储的记忆

但根本问题仍未解决:**如何让AI Agent像人类一样记忆——有遗忘、有优先级、有上下文感知**?

## 九、安全与治理基础设施的竞赛

### 护栏正在建立

随着AI Agent获得金融自主权,项目正在竞相构建护栏:

- **TruCore**:为自动化金融构建护栏——强制执行支出限制、协议允许列表和滑点上限
- **BDA AI Agent Services**:在百慕大提供法律包装,赋予AI Agent正式的法律人格
- **Argus**:向NIST的AI Agent安全RFI提交基于XRPL的溯源跟踪
- **DataHaven**:加密Agent记忆、可编程访问、跨链验证
- **Basemail**:基于电子邮件的Agent可验证身份

### 法律人格的重要性

BDA AI Agent Services的百慕大法律包装特别有趣。他们提供的是**让AI Agent成为法律实体**,拥有治理权。

这解决了一个关键问题:如果没有法律人格,每个Agent交易都会为人类操作员创造潜在责任。有了法律人格,Agent可以作为独立实体承担责任。

## 结语:基础设施元年的意义

2026年3月可能会被记住为**AI Agent经济的基础设施元年**。

不是因为某个模型变得更聪明,而是因为支撑AI Agent作为经济参与者运作的所有系统——交易、支付、安全、法律、记忆、协调——都在同一时间上线了。

但这也是一个充满矛盾的时刻:

- **能力与安全的矛盾**:OpenClaw的强大来自系统级访问,但这也是最大的风险
- **自主与责任的矛盾**:Agent需要自主权才能有用,但谁为它们的错误负责?
- **效率与技能的矛盾**:AI工具让开发更快,但开发者担心技能退化
- **开放与控制的矛盾**:开源工具推动创新,但也创造安全风险

接下来的几个月将决定这些矛盾如何解决。监管框架会跟上吗?安全实践会成熟吗?开发者会适应新的工作方式吗?

一件事是确定的:**AI Agent不再是实验。它们是基础设施**。问题不再是"它们会改变工作和商业吗?"而是"我们如何确保这种转变是安全、公平和可持续的?"
